🔌 أوامر ADB الثلاثون — شرح تفصيلي ممل

الأمر ١: adb devices — عرض الأجهزة المتصلة

$ adb devices 
# 📌 ماذا يفعل: يبحث عن هاتف أندرويد موصول بالكمبيوتر عبر USB وفي وضع تصحيح USB.
# يبدأ خادم ADB تلقائياً، يمسح المنافذ، يرسل إشارة استفسار، ويعرض الرقم التسلسلي مع الحالة.

adb devices
adb devices -l   # تفاصيل إضافية (اسم الموديل، الرقم التسلسلي)

# 📋 النتيجة الصحيحة:
List of devices attached
ABC123DEF    device

# الحالات الممكنة:
# device       = متصل وجاهز ✅
# offline      = متصل لكن لا يستجيب ← أعد تشغيل الهاتف أو adb kill-server
# unauthorized = لم توافق على تصحيح USB ← افتح الهاتف واضغط Allow
# recovery     = الجهاز في Recovery Mode
# fastboot     = الجهاز في Fastboot Mode

# ❌ أشهر الأخطاء:
# ١. قائمة فارغة → الكابل شحن فقط، أو تصحيح USB غير مفعل
# ٢. 'adb' not recognized → ADB غير مثبت أو PATH غير مضاف
# ٣. server version mismatch → نسختان من ADB (احذف القديمة)

# 🔧 الحل السحري:
adb kill-server && adb start-server && adb devices

الأمر ٢: adb shell — فتح نافذة أوامر داخل الهاتف

$ adb shell 
# 📌 ماذا يفعل: يفتح جلسة طرفية داخل الهاتف. كل أمر تكتبه ينفذ على معالج الهاتف.
# المؤشر يتغير من C:\Users\...> إلى اسم_الجهاز:/ $

adb shell
bullhead:/ $   ← $ = مستخدم عادي | # = صلاحيات Root

# تنفيذ أمر واحد بدون دخول Shell:
adb shell ls /sdcard

# ❌ أشهر خطأ: error: more than one device → حدد: adb -s ABC123DEF shell
# للخروج: اكتب exit أو اضغط Ctrl+D

الأمر ٣: adb shell ls — استعراض ملفات الهاتف

$ adb shell ls /sdcard 
# 📌 ماذا يفعل: يعرض محتويات مجلد في الهاتف. مثل dir في ويندوز.

adb shell ls /sdcard/              # الذاكرة الداخلية
adb shell ls /sdcard/Download/    # مجلد التحميلات
adb shell ls /system/              # مجلد النظام (يحتاج Root)
adb shell ls -la /sdcard/          # عرض مفصل مع الصلاحيات

الأمر ٤: adb push — رفع ملف للهاتف

$ adb push [كمبيوتر] [هاتف] 
# 📌 ماذا يفعل: ينقل ملفاً أو مجلداً من الكمبيوتر إلى الهاتف.

adb push C:\file.zip /sdcard/Download/
adb push C:\photos\ /sdcard/DCIM/

# ❌ أشهر خطأ: cannot stat → الملف غير موجود في الكمبيوتر

الأمر ٥: adb pull — سحب ملف من الهاتف

$ adb pull [هاتف] [كمبيوتر] 
# 📌 ماذا يفعل: يسحب ملفاً أو مجلداً من الهاتف إلى الكمبيوتر.

adb pull /sdcard/DCIM/photo.jpg C:\Desktop\
adb pull /sdcard/DCIM/ C:\backup\          # سحب مجلد كامل

# ❌ أشهر خطأ: remote object does not exist → المسار خطأ في الهاتف

الأمر ٦: adb install — تثبيت تطبيق

$ adb install app.apk 
# 📌 ماذا يفعل: يثبت تطبيق APK على الهاتف مباشرة من الكمبيوتر.

adb install app.apk              # تثبيت عادي
adb install -r app.apk           # إعادة تثبيت (استبدال النسخة الموجودة)
adb install -d app.apk           # السماح بتخفيض الإصدار
adb install -s app.apk           # تثبيت على التخزين الخارجي

# 📋 النتيجة: Success = تم ✅
# ❌ INSTALL_FAILED_INSUFFICIENT_STORAGE = مساحة غير كافية
# ❌ INSTALL_FAILED_UPDATE_INCOMPATIBLE = استخدم -r

الأمر ٧: adb uninstall — إلغاء تثبيت تطبيق

$ adb uninstall com.package.name 
# 📌 ماذا يفعل: يزيل تطبيقاً بالكامل. تحتاج اسم الحزمة وليس اسم التطبيق.

# كيف تعرف اسم الحزمة؟
adb shell pm list packages | findstr "whatsapp"
package:com.whatsapp

adb uninstall com.whatsapp        # إلغاء تثبيت
adb uninstall -k com.whatsapp     # إلغاء مع الاحتفاظ بالبيانات

الأمر ٨: adb logcat — سجل النظام للتشخيص

$ adb logcat — أقوى أداة تشخيصية 
# 📌 ماذا يفعل: يعرض سجل النظام. كل خطأ، تحذير، وحدث يسجل هنا.

adb logcat -d > log.txt           # حفظ السجل في ملف
adb logcat *:E                    # الأخطاء فقط
adb logcat *:W                    # التحذيرات والأخطاء
adb logcat -s BatteryService      # تصفية حسب Tag
adb logcat -v time                # مع الوقت والتاريخ
adb logcat -c                     # مسح السجل القديم

# ابحث في الملف عن: FATAL EXCEPTION, ANR, OutOfMemoryError, NullPointerException

الأمر ٩: adb reboot — إعادة تشغيل الهاتف

$ adb reboot [نوع] 
# 📌 ماذا يفعل: يعيد تشغيل الهاتف برمجياً (Soft Reboot).

adb reboot              # إعادة تشغيل عادية
adb reboot bootloader   # إلى Fastboot
adb reboot recovery     # إلى Recovery
adb reboot download     # لسامسونج: Download Mode

# 🔧 إذا لم يستجب الهاتف: اضغط زر الباور ١٠-١٥ ثانية

الأمر ١٠: adb reboot bootloader — الدخول إلى Fastboot

$ adb reboot bootloader 
# 📌 ماذا يفعل: يعيد تشغيل الهاتف إلى وضع Bootloader/Fastboot.
# هذا الوضع للصيانة العميقة — تفليش أقسام، فتح Bootloader، إلخ.
# ⚠️ سامسونج لا تستخدم Fastboot. استخدم adb reboot download + Odin.

# للتأكد أنك في Fastboot:
fastboot devices
ABC123DEF    fastboot

# للخروج:
fastboot reboot

الأمر ١١: adb reboot recovery — الدخول إلى Recovery

$ adb reboot recovery 
# 📌 ماذا يفعل: يعيد تشغيل الهاتف إلى وضع Recovery Mode.
# بيئة صيانة مستقلة تسمح بمسح البيانات، مسح الكاش، وتفليش التحديثات.
# مفيد عندما أزرار الصوت معطلة أو الشاشة مكسورة.

# ❌ مشكلة "No Command" (روبوت ميت):
# اضغط زر الباور ثم اضغط زر رفع الصوت مرة واحدة (اختصار سري)

الأمر ١٢: adb backup — نسخ احتياطي كامل

$ adb backup -apk -shared -all -f backup.ab 
# 📌 ماذا يفعل: ينشئ نسخة احتياطية كاملة للجهاز. سيطلب تأكيداً على الهاتف.

adb backup -apk -shared -all -f backup.ab

# -apk    = يشمل ملفات التطبيقات
# -shared = يشمل التخزين المشترك (صور، فيديوهات)
# -all    = كل التطبيقات
# -f      = اسم الملف الناتج

# ⚠️ هذه الطريقة قديمة ولا تشمل كل البيانات. للنسخ الكامل استخدم TWRP أو dd.

الأمر ١٣: adb restore — استعادة نسخة احتياطية

$ adb restore backup.ab 
# 📌 ماذا يفعل: يستعيد نسخة احتياطية تم إنشاؤها بـ adb backup.
# سيطلب تأكيداً على الهاتف.

adb restore backup.ab

الأمر ١٤: adb sideload — تفليش روم عبر Recovery

$ adb sideload rom.zip 
# 📌 ماذا يفعل: يفلش ملف ZIP على الهاتف من خلال Recovery Mode.
# يجب أن يكون الهاتف في Recovery واختيار "Apply update from ADB".

# الخطوات:
# ١. adb reboot recovery
# ٢. في الهاتف: اختر Apply update from ADB
# ٣. في الكمبيوتر:
adb sideload rom.zip

# تظهر نسبة التقدم على الجهازين معاً
# ❌ protocol fault → الهاتف ليس في وضع Sideload

الأمر ١٥: adb shell pm list packages — عرض التطبيقات

$ adb shell pm list packages 
# 📌 ماذا يفعل: يعرض كل التطبيقات المثبتة على الهاتف.
# مفيد للبحث عن تطبيقات خبيثة أو معرفة اسم الحزمة.

adb shell pm list packages                    # كل التطبيقات
adb shell pm list packages -3                # تطبيقات المستخدم فقط
adb shell pm list packages -s                # تطبيقات النظام فقط
adb shell pm list packages -f                # مع مسار ملف APK
adb shell pm list packages | grep "facebook"  # بحث

الأمر ١٦: adb shell pm disable-user — تعطيل تطبيق

$ adb shell pm disable-user com.bloatware 
# 📌 ماذا يفعل: يعطل تطبيقاً بدون الحاجة لـ Root. التطبيق يختفي ويتوقف.
# الفرق بين disable و uninstall: disable يخفي التطبيق فقط، uninstall يحذفه.

adb shell pm disable-user com.bloatware      # تعطيل
adb shell pm enable com.bloatware            # إعادة تفعيل
adb shell pm uninstall -k --user 0 com.bloatware  # حذف للمستخدم الحالي

# ⚠️ احذر: تعطيل تطبيقات نظام حرجة قد يسبب Bootloop!

الأمر ١٧: adb shell screencap — لقطة شاشة

$ adb shell screencap -p /sdcard/screen.png 
# 📌 ماذا يفعل: يأخذ لقطة شاشة (Screenshot) ويحفظها في الهاتف.

adb shell screencap -p /sdcard/screen.png
adb pull /sdcard/screen.png              # اسحبها للكمبيوتر

# 🔧 مفيد لتوثيق المشكلة للعميل أو للرجوع إليها.

الأمر ١٨: adb shell screenrecord — تسجيل شاشة

$ adb shell screenrecord /sdcard/video.mp4 
# 📌 ماذا يفعل: يسجل فيديو للشاشة. اضغط Ctrl+C للإيقاف.

adb shell screenrecord /sdcard/video.mp4
# اضغط Ctrl+C للإيقاف، ثم:
adb pull /sdcard/video.mp4

# خيارات إضافية:
# --time-limit 30 = مدة ٣٠ ثانية
# --bit-rate 4000000 = جودة 4Mbps

الأمر ١٩: adb shell dumpsys — معلومات النظام

$ adb shell dumpsys [خدمة] 
# 📌 ماذا يفعل: يعرض معلومات تفصيلية عن أي خدمة في النظام.
# هذا كنز معلومات لفني الصيانة.

adb shell dumpsys battery              # حالة البطارية
adb shell dumpsys wifi                 # معلومات الواي فاي
adb shell dumpsys meminfo             # استهلاك الرام
adb shell dumpsys cpuinfo             # استهلاك المعالج
adb shell dumpsys telephony.registry  # معلومات الشبكة
adb shell dumpsys sensorservice       # كل الحساسات
adb shell dumpsys media.camera        # مواصفات الكاميرات
adb shell dumpsys audio               # حالة الصوت والمايك

الأمر ٢٠: adb shell dumpsys battery — فحص البطارية

$ adb shell dumpsys battery 
# 📌 أهم أمر لتشخيص البطارية قبل فتح الهاتف.

adb shell dumpsys battery

# 📋 القيم المهمة وكيف تقرأها:
# health: 2 = سليمة ✅
# health: 3 = حرارة زائدة 🔥
# health: 4 = تالفة ❌ (تحتاج استبدال)
# health: 5 = جهد زائد ⚡ (خطر!)
# level: 85 = نسبة الشحن 85%
# temperature: 350 = 35°C (طبيعي)
# temperature: 450+ = خطر! افصل الشاحن فوراً
# technology: Li-ion = نوع البطارية

الأمر ٢١: adb shell getprop — خصائص النظام

$ adb shell getprop 
# 📌 ماذا يفعل: يقرأ خصائص النظام. يعرض مئات المعلومات عن الهاتف.

adb shell getprop ro.product.model              # اسم الموديل
adb shell getprop ro.build.version.release     # إصدار أندرويد
adb shell getprop ro.build.version.sdk         # API Level
adb shell getprop ro.product.cpu.abi           # نوع المعالج
adb shell getprop ro.product.manufacturer      # الشركة المصنعة
adb shell getprop gsm.version.baseband         # إصدار Baseband
adb shell getprop ro.boot.flash.locked         # حالة Bootloader (1=مقفول)
adb shell getprop ro.serialno                  # الرقم التسلسلي

# لعرض كل الخصائص دفعة واحدة:
adb shell getprop

الأمر ٢٢: adb shell setprop — تعديل خصائص (Root)

$ adb shell setprop [خاصية] [قيمة] 
# 📌 ماذا يفعل: يعدل خاصية من خصائص النظام. يتطلب Root في معظم الحالات.
# التعديل مؤقت — يزول بعد إعادة التشغيل.

adb shell setprop debug.hwui.renderer skiagl   # تغيير محرك الرسوميات
adb shell setprop persist.sys.usb.config mtp   # تفعيل وضع MTP

# ⚠️ بحذر! تعديل خاطئ قد يسبب Bootloop.

الأمر ٢٣: adb shell am start — تشغيل الأنشطة

$ adb shell am start 
# 📌 ماذا يفعل: يشغل أي Activity في النظام — تطبيقات، صفحات إعدادات، نوايا.

adb shell am start -a android.intent.action.VIEW -d "https://google.com"
# يفتح الرابط في المتصفح

adb shell am start -a android.settings.WIFI_SETTINGS
# يفتح إعدادات الواي فاي

adb shell am start -a android.settings.APPLICATION_DETAILS_SETTINGS -d "package:com.app"
# يفتح إعدادات تطبيق محدد

adb shell am start -a android.intent.action.MAIN -c android.intent.category.HOME
# يعيد تشغيل الواجهة الرئيسية — حل لتجمد الشاشة

الأمر ٢٤: adb shell input — محاكاة اللمس والكتابة

$ adb shell input 
# 📌 ماذا يفعل: يحاكي إدخالات المستخدم. مفيد عندما الشاشة مكسورة.

adb shell input tap 500 500          # لمس النقطة (500,500)
adb shell input swipe 500 1000 500 500  # سحب للأعلى
adb shell input text "hello"         # كتابة نص
adb shell input keyevent 26          # زر الباور
adb shell input keyevent 3           # زر Home
adb shell input keyevent 4           # زر Back
adb shell input keyevent 66          # زر Enter

# 🔧 يمكنك فتح قفل الشاشة بهذه الأوامر!

الأمر ٢٥: adb shell wm size — دقة الشاشة

$ adb shell wm 
# 📌 ماذا يفعل: يعرض أو يغير دقة الشاشة وكثافتها.

adb shell wm size                     # عرض الدقة الحالية
adb shell wm density                  # عرض الكثافة الحالية
adb shell wm size 1080x1920           # تغيير الدقة (مؤقت)
adb shell wm size reset               # إعادة الدقة الأصلية
adb shell wm density 440              # تغيير الكثافة

# 🔧 مفيد لحل مشاكل الشاشة بعد تغييرها. ⚠️ قيم خاطئة = شاشة سوداء.

الأمر ٢٦: adb shell settings — تعديل إعدادات النظام

$ adb shell settings 
# 📌 ماذا يفعل: يعدل إعدادات النظام مباشرة من سطر الأوامر.
# settings list = عرض | settings put = تعديل | settings get = قراءة

adb shell settings put global airplane_mode_on 1    # تفعيل طيران
adb shell settings put global adb_enabled 0         # تعطيل ADB
adb shell settings put system screen_brightness 128 # سطوع 50%
adb shell settings put system show_touches 1        # إظهار نقاط اللمس
adb shell settings get global airplane_mode_on      # قراءة حالة الطيران

# 🔧 مفيد لإصلاح مشاكل الإعدادات بدون فتح الهاتف.

الأمر ٢٧: adb kill-server / start-server — إدارة خادم ADB

$ adb kill-server && adb start-server 
# 📌 kill-server: يوقف خادم ADB تماماً ويقطع كل الاتصالات.
# 📌 start-server: يشغل خادم ADB من جديد.

adb kill-server
adb start-server

# الحل السحري لمشاكل ADB — ينظف أي تضارب أو اتصال عالق:
adb kill-server && adb start-server && adb devices

# 🔧 استخدمها عندما:
# - الهاتف لا يظهر في adb devices
# - تظهر رسالة "server version mismatch"
# - الهاتف يظهر offline
# - تريد "تنظيف" الاتصالات القديمة

الأمر ٢٨: adb connect — اتصال ADB لاسلكي عبر WiFi

$ adb connect [IP]:5555 
# 📌 ماذا يفعل: يتيح استخدام ADB عبر WiFi بدون كابل USB.
# مفيد جداً عندما يكون بورت الشحن تالفاً.

# الإعداد الأولي (يحتاج كابل USB مرة واحدة فقط):
adb tcpip 5555                       # تفعيل وضع WiFi على الهاتف
# الآن افصل الكابل
adb connect 192.168.1.10:5555        # استخدم IP الهاتف

# لمعرفة IP الهاتف:
# الإعدادات > حول الهاتف > الحالة > عنوان IP
# أو: adb shell ip addr show wlan0

# للعودة للوضع السلكي:
adb usb

# ❌ أشهر خطأ: connection refused → الهاتف والكمبيوتر ليسا على نفس الشبكة

الأمر ٢٩: adb shell su — صلاحيات Root

$ adb shell su 
# 📌 ماذا يفعل: يرفع صلاحياتك إلى Root (Superuser).
# يتطلب أن يكون الهاتف عليه Root (Magisk مثلاً).

adb shell
bullhead:/ $ su                    # اطلب صلاحيات Root
bullhead:/ #                        # # = أنت الآن Root!

# تنفيذ أمر واحد كـ Root بدون دخول Shell:
adb shell su -c "ls /data"

# ⚠️ بحذر شديد! مع Root يمكنك تدمير النظام بالكامل.
# لا تستخدم rm -rf أو dd إلا إذا كنت متأكداً 100%.

الأمر ٣٠: adb shell dd — نسخ واستعادة الأقسام

$ adb shell dd — أهم أمر للنسخ الاحتياطي 
# 📌 ماذا يفعل: ينسخ بيانات خام (Raw) بين الأقسام والملفات.
# هذا هو الأمر الوحيد الذي يحفظ EFS و IMEI بشكل صحيح. يتطلب Root.

# ⚠️⚠️⚠️ تحذير: dd إذا أخطأت (عكست if و of) قد تدمر الهاتف للأبد!

# نسخ قسم EFS (يحتوي على IMEI):
adb shell su -c "dd if=/dev/block/bootdevice/by-name/efs of=/sdcard/efs.img"
adb pull /sdcard/efs.img              # احفظه في مكان آمن!

# نسخ أقسام أخرى:
adb shell su -c "dd if=/dev/block/bootdevice/by-name/persist of=/sdcard/persist.img"
adb shell su -c "dd if=/dev/block/bootdevice/by-name/boot of=/sdcard/boot.img"
adb shell su -c "dd if=/dev/block/bootdevice/by-name/recovery of=/sdcard/recovery.img"

# استعادة:
adb push efs.img /sdcard/
adb shell su -c "dd if=/sdcard/efs.img of=/dev/block/bootdevice/by-name/efs"
adb reboot

# if = input file (المصدر) | of = output file (الوجهة)
# ⚠️ لا تعكسهم أبداً! if = تقرأ منه | of = تكتب فيه

⚡ أوامر Fastboot الخمسة عشر — شرح تفصيلي

⚠️
تذكير: سامسونج لا تستخدم Fastboot. استخدم Download Mode + برنامج Odin. جميع الماركات الأخرى تستخدم Fastboot.

الأمر ٣١: fastboot devices — عرض الأجهزة

$ fastboot devices

fastboot devices

الأمر ٣٢: fastboot reboot — إعادة تشغيل

$ fastboot reboot 
# 📌 يعيد تشغيل الهاتف من وضع Fastboot إلى النظام العادي.

fastboot reboot              # إعادة تشغيل عادية
fastboot reboot recovery     # إعادة تشغيل إلى Recovery
fastboot reboot bootloader   # إعادة تشغيل إلى Fastboot مرة أخرى

الأمر ٣٣: fastboot flash — تفليش قسم

$ fastboot flash [قسم] [ملف.img] 
# 📌 يكتب ملف .img مباشرة على قسم من أقسام الذاكرة.
# هذا هو الأمر الأساسي لتفليش الرومات وإصلاح مشاكل الإقلاع.

fastboot flash boot boot.img        # تفليش Kernel
fastboot flash recovery twrp.img    # تفليش Recovery مخصص
fastboot flash system system.img    # تفليش النظام
fastboot flash vendor vendor.img    # تفليش تعريفات الجهاز

# تفليش كل الأقسام دفعة واحدة:
fastboot flashall                   # ينفذ من مجلد factory image

# ❌ FAILED: Partition not found → اسم القسم خطأ
# ❌ FAILED: Flashing is not allowed → Bootloader مقفول

الأمر ٣٤: fastboot erase — مسح قسم

$ fastboot erase [قسم] 
# 📌 يمسح محتوى قسم بالكامل. يختلف عن format.
# erase = يمسح البيانات فقط | format = يعيد بناء نظام الملفات

fastboot erase userdata            # مسح بيانات المستخدم
fastboot erase cache               # مسح الكاش
fastboot erase system              # مسح النظام (خطر!)

# ⚠️ erase userdata يمسح كل تطبيقاتك وإعداداتك!

الأمر ٣٥: fastboot format — تهيئة قسم

$ fastboot format [قسم] 
# 📌 يهيئ قسماً بنظام ملفات محدد (ext4/f2fs). أعمق من erase.

fastboot format data               # تهيئة قسم data (ext4)
fastboot format:f2fs data          # تهيئة بنظام f2fs (أسرع)
fastboot format cache              # تهيئة قسم cache

# ⚠️ format data يمسح كل شيء بما فيها التخزين الداخلي (صور، فيديوهات)!

الأمر ٣٦: fastboot boot — تشغيل صورة مؤقتة

$ fastboot boot [ملف.img] 
# 📌 يشغل صورة (مثل TWRP) مؤقتاً بدون تفليشها دائماً.
# عند إعادة التشغيل، يختفي كل شيء ويعود الهاتف لحالته الأصلية.

fastboot boot twrp.img

# هذا آمن 100% — لا يغير أي شيء دائم في الهاتف.
# مثالي لتجربة Recovery قبل تثبيته، أو لعمل نسخة احتياطية سريعة.

الأمر ٣٧: fastboot oem unlock — فتح Bootloader (قديم)

$ fastboot oem unlock 
# 📌 يفتح Bootloader للأجهزة القديمة.
# ⚠️⚠️⚠️ هذا الإجراء يمسح كل بيانات الهاتف بالكامل!

fastboot oem unlock

# يستخدم مع: Samsung, HTC, Sony, Motorola القديمة
# بعد الأمر، سيظهر تأكيد على شاشة الهاتف. استخدم أزرار الصوت للتنقل.

الأمر ٣٨: fastboot flashing unlock — فتح Bootloader (جديد)

$ fastboot flashing unlock 
# 📌 يفتح Bootloader للأجهزة الحديثة.
# ⚠️⚠️⚠️ يمسح كل بيانات الهاتف بالكامل!

fastboot flashing unlock

# يستخدم مع: Google Pixel, OnePlus, Xiaomi, Nothing, Motorola الجديدة
# يجب تفعيل OEM Unlocking في خيارات المطور أولاً.

الأمر ٣٩: fastboot flashing lock — قفل Bootloader

$ fastboot flashing lock 
# 📌 يعيد قفل Bootloader للعودة للحالة الرسمية.
# ⚠️⚠️⚠️ يمسح كل بيانات الهاتف أيضاً!

fastboot flashing lock

# استخدم هذا قبل بيع الهاتف أو إعادته للضمان.
# تأكد من أن النظام روم رسمي قبل القفل، وإلا قد تحصل على Brick!

الأمر ٤٢: fastboot set_active — تبديل A/B Slot

$ fastboot set_active [a|b] 
# 📌 الأجهزة الحديثة لها نظامان (Slot A و Slot B) لتحديثات سلسة.
# إذا فشل تحديث على Slot، يمكنك التبديل للآخر.

fastboot getvar current-slot       # معرفة الـ Slot النشط حالياً
fastboot set_active a              # تفعيل Slot A
fastboot set_active b              # تفعيل Slot B
fastboot --set-active=other        # تفعيل الـ Slot الآخر تلقائياً

# 🔧 حل سحري لـ Bootloop بعد تحديث فاشل!
# إذا فشل التحديث على Slot A، انتقل لـ Slot B:
fastboot --set-active=other
fastboot reboot

الأمر ٤٣: fastboot --set-active=other

# يجرب النظام الآخر الذي قد يكون سليماً.

🔥 سكريبتات جاهزة للاستخدام

سكريبت ١: backup-all.sh — نسخ احتياطي كامل (Linux/macOS)

backup-all.sh 
#!/bin/bash
# نسخ احتياطي كامل قبل أي تفليش

DEVICE=$(adb shell getprop ro.product.model | tr -d '\r')
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="backup_${DEVICE}_${DATE}"

mkdir -p "$BACKUP_DIR"

echo "[*] Backing up partitions..."
adb shell su -c "dd if=/dev/block/bootdevice/by-name/boot of=/sdcard/boot_backup.img"
adb shell su -c "dd if=/dev/block/bootdevice/by-name/recovery of=/sdcard/recovery_backup.img"
adb shell su -c "dd if=/dev/block/bootdevice/by-name/efs of=/sdcard/efs_backup.img"

echo "[*] Pulling backups..."
adb pull /sdcard/boot_backup.img "$BACKUP_DIR/"
adb pull /sdcard/recovery_backup.img "$BACKUP_DIR/"
adb pull /sdcard/efs_backup.img "$BACKUP_DIR/"

echo "[✓] Backup complete: $BACKUP_DIR"
echo "[✓] Keep this folder safe! It contains your IMEI."

💎 نصائح ذهبية للفني المحترف

✅
قواعد لا تنساها أبداً:
  1. السوفتوير أولاً دائماً — مجاني وآمن. لا تفتح هاتفاً قبل أن تستنفد كل الحلول البرمجية.
  2. انسخ EFS قبل أي تفليش — هذه بوليصة التأمين الوحيدة لـ IMEI.
  3. التشخيص قبل التفكيك — ٥٠٪ من التشخيص يأتي من أسئلة العميل.
  4. كابل USB الأصلي — ٥٠٪ من مشاكل ADB سببها كابل شحن فقط.
  5. لا تفليش روم لطراز مختلف — قد تدمر IC الشبكة بشكل دائم.
  6. صوّر كل خطوة — عند تفكيك هاتف، صور كل شيء لتعرف كيف تعيده.
