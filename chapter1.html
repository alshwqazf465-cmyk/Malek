🔧 ما هو Bootloader؟

Bootloader هو أول برنامج يعمل عند تشغيل الهاتف. يتحقق من سلامة النظام ثم يُطلق النواة (Kernel). إذا كان Locked، لن تستطيع تثبيت ROMs غير رسمية.

💡
معلومة مهمة: فتح Bootloader يمسح جميع بيانات الجهاز تلقائياً كإجراء أمان. احفظ نسخة احتياطية قبل البدء!
Fastboot — فحص حالة Bootloader 
# التحقق من حالة القفل
$ fastboot oem device-info
(bootloader) Device unlocked: false
(bootloader) Device critical unlocked: false

# فتح Bootloader (سيُمسح كل شيء!)
$ fastboot flashing unlock
OKAY [  0.023s]
finished. total time: 0.023s

🔄 Recovery Mode

بيئة خاصة للصيانة. النوع الرسمي (Stock) يتيح إعادة ضبط المصنع وتحديثات OTA فقط. النوع المعدّل (Custom مثل TWRP) يتيح نسخ احتياطي كامل وتفليش ROMs.

مقارنة بين Recovery modes

1
Stock Recovery — رسمي من الشركة: Factory Reset + Apply Update from ADB
⬇
2
Custom Recovery (TWRP) — معدّل: Backup/Restore + Flash ZIP + File Manager + Terminal
⬇
3
Recovery Mode الدخول: Power + Volume Up (أو Home في سامسونج القديمة)

⚡ Fastboot Mode

وضع برمجي منخفض المستوى يتيح التفليش المباشر للأقسام. يعمل قبل إقلاع النظام الرئيسي.

Terminal
$ adb reboot bootloader # إعادة التشغيل إلى Fastboot $ fastboot devices # التحقق من الاتصال ABC123DEF fastboot $ fastboot flash boot boot.img # تفليش قسم boot Sending 'boot' (131072 KB) OKAY [ 3.210s] Writing 'boot' OKAY [ 0.890s] Finished. Total time: 4.100s $ fastboot reboot # إعادة التشغيل

🔌 ADB (Android Debug Bridge)

جسر بين الكمبيوتر والهاتف. يتيح تنفيذ أوامر Shell، نقل ملفات، تثبيت تطبيقات، وحتى الوصول للهاتف المعطّل جزئياً.

ADB — الأوامر الأساسية 
# التحقق من الاتصال
$ adb devices
List of devices attached
ABC123DEF    device

# الدخول إلى Shell
$ adb shell
bullhead:/ $

# تثبيت APK
$ adb install app.apk

# سحب ملف من الهاتف
$ adb pull /sdcard/photo.jpg ./photo.jpg

# دفع ملف إلى الهاتف
$ adb push file.zip /sdcard/Download/

# إعادة التشغيل إلى Recovery
$ adb reboot recovery

# إعادة التشغيل إلى Fastboot
$ adb reboot bootloader

# تسجيل Logcat (للتشخيص)
$ adb logcat -d > log.txt

📦 ROM / Firmware

المصطلح الشرح الملفات
Stock ROM النظام الرسمي من الشركة .tar, .tar.md5, .zip
Custom ROM نظام معدّل (LineageOS, Pixel Experience) .zip
Firmware ROM + Modem + Bootloader + Recovery مجلد كامل
Factory Image صورة المصنع الكاملة .img لكل قسم

📡 Modem / Baseband

الجزء المسؤول عن الاتصال الخلوي. إذا فُقد Baseband: لا شبكة، لا مكالمات، لا إنترنت. يُخزّن في قسم modem أو radio.

⚠️
تحذير: تفليش ملف Modem خاطئ لطراز مختلف قد يُدمر IC الشبكة بشكل دائم!
التحقق من Baseband 
# عبر ADB Shell
$ adb shell getprop | grep baseband
[gsm.version.baseband]: [M8992_20210.05.05.21R]

# أو عبر الإعدادات
# Settings > About Phone > Baseband version

# إذا ظهر "Unknown" — المشكلة في Modem/EFS

🔐 EFS (Encrypting File System)

قسم حساس يحتوي على معلومات الجهاز الفريدة:

  • IMEI (رقم الهوية)
  • معايرة الشبكة (Calibration)
  • WiFi MAC Address
  • Bluetooth Address
🚨
خطر! تلف EFS يعني فقدان IMEI للأبد إذا لم يكن لديك نسخة احتياطية. احفظ نسخة EFS قبل أي تفليش!
نسخ احتياطي واستعادة EFS 
# نسخ احتياطي (يتطلب Root)
$ adb shell su -c "dd if=/dev/block/bootdevice/by-name/efs of=/sdcard/efs_backup.img"
$ adb pull /sdcard/efs_backup.img ./efs_backup.img

# استعادة
$ adb push efs_backup.img /sdcard/
$ adb shell su -c "dd if=/sdcard/efs_backup.img of=/dev/block/bootdevice/by-name/efs"

📱 IMEI

رقم هوية الجهاز الفريد (15 رقم). يُخزّن في EFS وعلى شريحة IC منفصلة (eMMC/UFS). تغييره غير قانوني، لكن استعادته بعد فقدان EFS مسموح.

التحقق من IMEI 
# عبر الاتصال
$ adb shell service call iphonesubinfo 1
Result: Parcel( 0x00000000: '000000000000000' )

# أو عبر الكود في الهاتف
# *#06#

💾 Partitions (أقسام الذاكرة)

القسم الوظيفة الحجم التقريبي
boot Kernel + Ramdisk (initrd) 32-64 MB
system نظام Android + التطبيقات المثبتة مسبقاً 2-4 GB
vendor تعريفات الشركة (Samsung, Xiaomi...) 500 MB - 1 GB
data بيانات المستخدم + التطبيقات + الإعدادات باقي المساحة
cache ذاكرة مؤقتة للتحديثات 256-512 MB
recovery بيئة الاسترداد 32-64 MB
bootloader محمّل الإقلاع (SPL/ABOOT) 1-2 MB
modem برنامج الشبكة (Baseband) 64-128 MB
efs IMEI, WiFi, Bluetooth MAC 16-32 MB
misc إعدادات Boot (مثل recovery command) 1 MB
عرض جميع الأقسام 
# عبر ADB (يتطلب Root)
$ adb shell ls /dev/block/bootdevice/by-name/
boot      modem    system    userdata
recovery  cache    vendor    efs
misc      persist  frp       sec

# معلومات تفصيلية
$ adb shell cat /proc/partitions
major minor  #blocks  name
 259        0   33554432 mmcblk0
 259        1      32768 mmcblk0p1  boot

📊 مخطط تدفق إقلاع Android

1
Power ON — الضغط على زر التشغيل
⬇
2
Boot ROM (PBL) — شريحة صلبة في المعالج، تتحقق من Bootloader
⬇
3
Bootloader (SBL/ABOOT) — يتحقق من توقيع Kernel
⬇
4
Kernel — يُحمّل نظام Android الأساسي
⬇
5
init — أول عملية في Android، تُطلق Zygote
⬇
6
Zygote — يُطلق System Server وكل التطبيقات
⬇
7
System Server — يُطلق كل خدمات Android (ActivityManager, WindowManager...)
⬇
8
Launcher — الشاشة الرئيسية تظهر! ✅
